# Website Readiness Review

Use this checklist after the visual implementation loop.

- The site has a real header, footer, title, description, canonical URL, and social preview image.
- The first screen makes the product obvious.
- The primary CTA points to a real working surface.
- The install command is accurate and verified.
- The GitHub, npm, docs, security, license, and issue links work.
- The page works on mobile and desktop.
- Text does not overlap or overflow.
- Generated illustrations are real assets, not screenshots pretending to be UI.
- Capture forms are low-friction and verified end to end.
- The page does not claim hosted services, pricing, telemetry, integrations, community, or support paths that do not exist.

Write a gap report before making changes. Commit after each readiness pass.
