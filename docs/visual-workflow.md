# Visual Workflow

Launchframe treats brand/site work as a visual process, not a text prompt exercise.

## Cycle 1: Mood Board And Artboards

1. Gather 20-40 reference screenshots that look excellent for the product category.
2. Save the actual images, not just text links.
3. Ask the visual model to extract visual DNA from the mood board.
4. Generate many candidate first-fold concepts.
5. Pick a direction and continue it fold by fold.
6. For continuations, preserve overlap: the next generation should see the previous lower half and continue from it.
7. Name each fold by purpose, such as hero, problem/solution, mechanism, integrations, proof, CTA.

Avoid over-determining the look in prose. The mood board should lead.

## Cycle 2: Implementation

Build the site from the selected artboards as a real website, not as stitched slides.

The implementation agent should:

- Inspect the artboards.
- Extract usable raster assets for illustrations.
- Build UI elements as HTML/CSS where appropriate.
- Preserve the selected visual system.
- Produce screenshots.
- Write a gap report after each pass.
- Commit each meaningful cycle.

## Cycle 3: Website Readiness

After visual fidelity is close, run a separate real-site pass:

- Header/footer/navigation.
- Metadata and social preview.
- Responsive layout.
- Links and CTAs.
- Accessibility basics.
- No overlapping text.
- No dead placeholder links.
- Capture path works if present.

This cycle should make the site real without redesigning the visual direction.
