# Local AI Desktop Readiness

Use when a desktop app records audio/video, transcribes locally, runs local AI models, stores local notes, or makes local-first/privacy claims.

## Scope

- Platforms:
- Audio/video capture permissions:
- Model name/source:
- Model license:
- Model distribution: bundled / downloaded / user-provided / none
- Transcript/note storage path:
- Network access during recording/transcription:

## Checks

- [ ] Microphone/screen/audio permission prompts are documented.
- [ ] Audio file retention, deletion, and export behavior are verified.
- [ ] Transcript and note storage paths are documented.
- [ ] Model source, license, redistribution rights, and update behavior are recorded.
- [ ] Offline recording/transcription proof is recorded or local/offline claims are weakened.
- [ ] Network egress during recording/transcription is verified by source inspection or runtime observation.
- [ ] Crash reporting, update checks, analytics, telemetry, and external APIs are absent, disclosed, or out of scope.
- [ ] OS permission prompts and privacy labels/disclosures are prepared where relevant.
- [ ] "Local", "private", "no telemetry", "encrypted", or "AI runs locally" claims are evidence-backed or owner-approved, or removed.
